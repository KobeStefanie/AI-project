# 灵感收集器（碎碎念）需求文档

> 版本 v1.0.0 | 2026-05-29 | 规划阶段 | 需求计划 v7

## 项目概述

**想法捕手**——不是备忘录。灵感往往是一段话、一个画面、一段声音或一个尚未深化的 idea，一闪而过，需要在消失前立刻抓住。输入摩擦要尽量降到最低，分类、标签、心情、主题建议由 AI 在事后异步处理。

**核心概念**：
- **灵感碎片**——单条捕获内容，保持独立，记录时间、来源、心情、媒体和上下文
- **主题**——非破坏性的上层容器，用于聚合相关碎片；碎片仍保持独立，不因加入主题而丢失原始语境
- **Inbox**——`status = raw` 的待处理视图，不是单独的数据状态

**核心工作流**：

```text
iPhone 零摩擦捕获 → 本地/服务端保存原始内容 → 后台 AI 初整理 → Windows 工作台整理 → 主题沉淀 / Markdown 扩展 / 日历回顾
```

**v1 一句话定位**：

> 支持文字、语音、图片、视频的跨端个人灵感捕获工具；iPhone 负责快速收集，Windows 负责 Inbox 整理、主题沉淀、日历回顾和时间线漫游；AI 负责后台初步整理，但 v1 不做语义搜索、提醒、AI 修正学习、视频理解和去重合并。

## §0.0 v1 范围边界

### §0.1 v1 必做

| 模块 | v1 范围 |
|------|---------|
| 文字捕获 | iPhone / Windows 均可创建文字灵感 |

| 功能 | 处理方式 |
|------|----------|

| AI 修正学习 | 延后到 v1.1 / v2 |

## §1.0 双端职责

| 维度 | iPhone（捕获端） | Windows（整理端） |
|------|------------------|-------------------|
| 定位 | 3 秒开记，零决策，尽量不打断当前场景 | 回顾、筛选、整理、主题沉淀、扩展深化 |
| 输入 | 文字、语音、拍照/图片、视频，可配上下文 | 键盘输入、Markdown 编辑、媒体查看 |
| 语音 | 录音优先保存；转写可用则补充，不阻塞提交 | 回放音频、查看/校对转写文本 |
| 图片 | 拍照或相册选择，多图上传 | 查看图片，作为灵感素材 |
| 视频 | 录制或相册选择，最多 1 个视频 | 播放视频，查看文件信息 |
| 首页 | 捕获输入区 + 语音/相机/视频快捷按钮 | 工作台：首页概览 + Inbox + 小日历 + 随机回顾 |
| 操作 | 捕获、查看、快速归档 | 筛选、批处理、主题聚合、扩展、状态流转 |
| 网络 | 支持离线轻量捕获；视频离线依赖系统相册 | 在线为主 |

## §2.0 功能清单

### §2.1 iPhone 端——快速捕获

- 打开直接显示输入区，自动聚焦，减少选择成本
- 按钮组：**语音**（麦克风）+ **拍照/选图**（相机）+ **视频** + **上下文标签**
- 上下文快捷标签（可选）：播客、聊天、读书、做梦、开会、路上、项目中、其他
- 默认捕获原则：先保存，后整理；AI 和转写不阻塞保存
- 文字：输入后立即提交保存
- 语音：点击开始 → 说话 → 点击结束 → 保存原始音频 → 可用时补充转写
- 图片：拍照或相册选取，可多张，可配文字或语音说明
- 视频：录制或相册选择，最多 1 个，单个 < 1GB，时长 < 10 分钟，可配文字或语音说明
- 提交后立即显示保存反馈：已保存 / 同步中 / AI 分析中
- 底部 Tab：**捕获** | **列表** | **设置**

### §2.2 iPhone 端——离线策略

- 文字：进入本地离线队列，联网后自动同步
- 音频：可进入 IndexedDB 离线队列；若存储失败，需要明确提示
- 图片：可进入 IndexedDB 离线队列；需要限制数量和大小，避免占满浏览器存储
- 视频：**不进入浏览器离线队列**；离线录制的视频保存在手机系统相册
- 离线视频补传流程：

```text
离线时录制视频到手机相册
↓
在灵感收集器中保存文字/语音说明
↓
该灵感标记 mediaStatus = pending
↓
联网后打开该灵感
↓
点击“补传视频”
↓
从相册选择视频并上传
↓
系统将视频关联到该灵感
```

### §2.3 iPhone 端——列表页

- 顶部搜索框：v1 支持关键词搜索
- 倒序卡片流
- 每条显示：内容摘要、AI 标签、关键词、心情图标、状态标记、时间戳、来源图标、主题归属
- 左滑：归档 / 删除
- 点击展开详情：播放语音、查看图片/视频、查看所属主题
- 可对 raw 灵感快速归档或放弃

### §2.4 随机回顾（两端共用）

- “随机漫步”——无目的地浏览过往灵感
- “随机 raw”——随机取一条未处理灵感，辅助清理 Inbox
- “主题回顾”——从某个主题中随机查看一条灵感
- 支持“再来一条”

### §2.5 iPhone 设置

- 同步状态
- 离线队列数量
- 存储用量
- AI 分析状态
- 媒体上传失败记录
- 语音转写开关 / 可用性提示

## §3.0 Windows 端设计

### §3.1 Windows 默认首页——工作台

Windows 默认首页不再是单独的日历热力图，而是“工作台”。工作台用于回答：今天有什么新灵感？还有多少没处理？下一步该整理什么？

布局建议：

```text
顶部：概览数据
左侧：导航
中间：Inbox / 最近待处理灵感
右侧：小日历热力图 + 随机回顾 + AI 主题推荐
```

顶部概览：
- 今日新增
- Inbox 待处理数量
- 本周新增
- 本月已扩展
- 媒体上传失败数量（如有）

主导航：
- 工作台
- Inbox
- 日历
- 主题
- 时间线
- 媒体
- 设置

### §3.2 Inbox / 列表整理模式

Inbox 是 `status = raw` 的视图，不是独立状态。

**左侧筛选与批处理**：
- 搜索框：v1 仅支持关键词搜索
- 状态快捷入口：全部 | Inbox(raw) | expanded | realized | archived | abandoned
- 状态、标签、心情、来源、主题筛选
- 日期范围筛选：今日 / 本周 / 本月 / 自定义范围
- 媒体类型筛选：文字 / 语音 / 图片 / 视频
- 多选后快速操作：批量归档、批量放弃、批量加入主题
- v1 不提供导出选中
- v1 不提供合并选中

**列表卡片显示**：
- AI 摘要 / 原文摘要
- 来源图标：文字 / 语音 / 图片 / 视频
- 标签、关键词、心情
- 状态、时间、设备来源
- 所属主题（可多个）
- AI 状态：分析中 / 已完成 / 失败
- 媒体状态：无媒体 / 上传中 / 已上传 / 上传失败 / 待补传

**右侧详情与编辑**：
- 原始内容 + 上下文来源
- 语音播放器 + 转写文本
- 图片查看
- 视频播放；显示文件名、大小、时长；v1 不显示封面图
- Markdown 扩展编辑（编辑 + 预览）
- 状态手动切换
- 标签和心情可手动修改，但 v1 不做 AI 修正学习
- 所属主题（可多个）：显示当前关联的多个主题，可加入或移除
- AI 推荐主题：用户确认后加入，不自动创建或加入
- 操作按钮：保存、归档、放弃、删除、加入主题、补传视频

### §3.3 日历视图

- 类似 GitHub 贡献图，按月/周展示
- 颜色深浅 = 当天灵感数量
- 悬停：日期、条数、主要关键词、主要心情
- 点击某天展开该日所有灵感列表
- v1 做轻量统计，不做完整 AI 分析面板

### §3.4 时间线视图

- 纵向时间轴，按月份分组
- 每条带状态色点标记
- 图片、语音、视频以媒体标识展示
- 视频可点击进入详情播放
- 支持按状态、标签、主题、媒体类型筛选

### §3.5 主题视图

**主题列表**：
- 卡片网格展示所有主题
- 每个卡片显示：主题色、标题、成员数量、最近活跃时间
- 点击进入主题详情页

**主题详情页**：
- 顶部：主题标题、描述、主题色，可编辑
- 主题级 Markdown 笔记区：写关于这个主题的整体思考
- 下方：该主题所有成员碎片的时间线列表
- 可手动加入新碎片、移除已有碎片
- AI 推荐：“还有 N 条碎片可能也属于这个主题，是否添加？”

**主题创建**：
- 手动创建：命名、选色、手动添加成员
- AI 建议创建：AI 只给建议，必须用户确认后创建
- 创建后碎片保持独立，通过 `ThemeInspiration` 关联表建立多对多主题关联
- 一条灵感可以同时属于多个主题

### §3.6 媒体视图

媒体视图是筛选入口，不是完整媒体库。

- 全部图片
- 全部语音
- 全部视频
- 上传失败 / 待补传
- 点击媒体后进入对应灵感详情

## §4.0 媒体规则

### §4.1 单条灵感媒体数量限制

| 媒体类型 | v1 限制 |
|----------|---------|
| 音频 | 最多 1 个主音频 |
| 图片 | 多张 |
| 视频 | 最多 1 个 |

### §4.2 视频规则

- 单个视频 < 1GB
- 视频时长 < 10 分钟
- 支持 iPhone 录制或从相册选择
- 支持 Windows 端播放
- v1 保存原始视频文件
- v1 不做服务端转码
- v1 不做视频剪辑
- v1 不生成视频封面图
- v1 不做视频 AI 识别
- v1 不做视频转写
- v1 不做视频摘要
- v1 不做视频内容理解
- v1 使用普通 multipart 上传
- 必须显示上传进度
- 上传前校验文件大小和视频时长
- 上传失败时保留已保存的灵感文本，不删除灵感
- 上传失败后允许重新选择文件并补传
- 分片上传延后到 v1.1 / v2

### §4.3 媒体存储方案

v1 媒体文件存服务端磁盘，SQLite 只保存媒体元数据和文件路径。

v1 部署目标为个人自用公网访问。服务器需准备足够磁盘空间；视频上传以单文件 `< 1GB` 为上限，具体请求体大小限制需要在应用服务、反向代理和部署平台中统一配置。

建议目录结构：

```text
uploads/
├── audio/
│   └── 2026/05/
├── images/
│   └── 2026/05/
└── videos/
    └── 2026/05/
```

数据库不直接保存二进制文件。

## §5.0 AI 自动处理

### §5.1 v1 AI 功能

AI 处理异步执行，不阻塞捕获保存。

| 功能 | 触发时机 | v1 实现 |
|------|----------|---------|
| 摘要 | 提交后后台处理 | 生成一句话摘要，用于列表显示 |
| 分类标签 | 提交后后台处理 | Claude API，根据文本/转写/说明生成 |
| 关键词提取 | 提交后后台处理 | Claude API，提取 3-5 个关键词 |
| 心情识别 | 提交后后台处理 | Claude API，识别 7 档心情 |
| 推荐已有主题 | 提交后后台处理 | 根据标题、摘要、关键词和已有主题判断 |
| 建议创建新主题 | 定期或提交后触发 | AI 给建议，用户确认后创建 |

AI 可分析的内容：
- 用户输入文字
- 语音转写文本（如果有）
- 图片/视频的文字说明
- 上下文标签
- 主题标题和主题笔记

AI 不分析的内容：
- 视频画面
- 视频声音
- 图片内容本身（v1 暂不做图片理解）

### §5.2 AI 状态

每条灵感需要记录 AI 处理状态：

```js
aiStatus: "pending" | "processing" | "done" | "failed"
aiError: string | null
processedAt: string | null
```

失败时允许手动重试。

## §6.0 语音输入

- v1 优先保证原始音频保存
- Web Speech API 可用时进行前端转写
- 转写结果写入 transcript
- 转写质量标记：high / medium / low / unknown
- 转写失败不影响灵感保存
- Windows 端可播放音频并手动校对转写文本
- Whisper 后备转写 v1 延后，仅保留后续接口设计空间

推荐流程：

```text
点击录音
↓
结束录音
↓
立即保存原始音频和文字说明
↓
后台/前端尝试转写
↓
成功后补充 transcript
↓
失败则显示“待校对”
```

## §7.0 状态、心情与搜索

### §7.1 灵感状态

```text
raw / expanded / realized / archived / abandoned
```

| 状态 | 含义 | 图标 |
|------|------|------|
| `raw` | 原始捕获，默认状态，属于 Inbox | ⚪ |
| `expanded` | 已扩展深化 | 🔵 |
| `realized` | 已实现 | ✅ |
| `archived` | 归档，有价值但暂不处理 | 📦 |
| `abandoned` | 放弃，认为价值较低 | ✕ |

状态不是严格线性流程。用户可以根据实际情况手动切换：

```text
raw → expanded
raw → realized
raw → archived
raw → abandoned
expanded → realized / archived / abandoned
```

### §7.2 心情分类（7 档）

| 值 | 标签 | 图标 |
|----|------|------|
| `excited` | 兴奋 | 😄 |
| `calm` | 平和 | 😐 |
| `low` | 低落 | 😔 |
| `confused` | 困惑 | 🤔 |
| `anxious` | 焦虑 | 😰 |
| `inspired` | 受启发的 | ✨ |
| `determined` | 坚定的 | 💪 |

### §7.3 搜索系统

v1 仅做关键词搜索。

搜索范围：
- content
- transcript
- expanded
- summary
- keywords
- tags
- context
- theme title

语义搜索、向量搜索、自然语言检索延后到 v1.1 / v2。

### §8.3 Theme

```js

  id: "uuid",
  title: "内容创作",
  description: "关于视频和写作的灵感合集",
  color: "#f97316",
  notes: "这个主题下的整体思考...",
  createdAt: "2026-05-29T12:00:00.000Z",
  updatedAt: "2026-05-29T18:00:00.000Z"
}
```

主题和灵感使用关联表。一条灵感可以属于多个主题，一个主题也可以包含多条灵感。不要在 `Inspiration` 中使用单个 `themeId` 字段，以避免把主题误设计成文件夹。

```js
ThemeInspiration {
  themeId: "uuid",
  inspirationId: "uuid",
  createdAt: "2026-05-29T12:00:00.000Z"
}
```

### §8.4 主题聚合 vs 合并

| | 主题聚合 | 去重合并 |
|------|----------|----------|
| v1 是否支持 | 支持 | 不支持 |
| 碎片独立性 | 保持独立 | 会破坏原始上下文 |
| 适用场景 | “这几条都和某个方向有关” | “同一条重复记录” |
| v1 处理方式 | 创建主题 / 加入主题 / 手动关联 | 延后，不提供入口 |

## §10.0 技术架构

| 层 | 技术 |
|----|------|
| 前端 | 单 HTML + Tailwind CSS CDN + Font Awesome + Vanilla JS |
| PWA | manifest.json + Service Worker |
| 后端 | Express.js (Node.js) |
| 数据库 | SQLite 抽象层；v1 本地 SQLite；后续可迁移 Turso |
| 媒体存储 | 服务端磁盘存储，数据库保存元数据和路径 |
| AI | Claude API，用于摘要、标签、关键词、心情、主题推荐 |
| 语音识别 | Web Speech API 可用则使用；Whisper 后备延后 |
| 离线 | LocalStorage / IndexedDB / Service Worker；视频离线保存在手机相册 |
| 访问方式 | 单人使用；v1 不做注册登录和多用户认证 |
| 部署 | 开发阶段 Windows 本机；v1 使用阶段按个人公网访问设计，可部署到 Node.js 服务器 / NAS / 云服务器 |

数据库策略：
- v1 使用本地 SQLite 文件
- 后端通过 db 抽象层访问数据库
- 业务代码避免直接绑定 SQLite 细节
- 数据模型和查询尽量保持 Turso 兼容
- 后续需要云端同步或更稳定远程访问时，可迁移到 Turso

安全策略：
- v1 不做注册登录
- v1 不做多用户认证
- 默认单人自用
- v1 使用阶段按个人公网访问设计，但不引入完整账号系统
- 公网部署必须通过部署层访问限制、反向代理、内网穿透鉴权或简单环境变量访问密钥保护服务，避免服务裸露

部署策略：
- 开发阶段：Windows 本机运行 Express + SQLite
- 手机测试：同一局域网访问 Windows 后端，或使用临时内网穿透
- v1 使用阶段：Node.js 服务器 / NAS / 云服务器任选其一
- 若使用反向代理，需要同步配置上传体积上限、请求超时和静态文件访问路径
