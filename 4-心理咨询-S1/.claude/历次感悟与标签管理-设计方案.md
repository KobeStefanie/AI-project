# 历次感悟与标签管理 - 设计方案

## 一、需求澄清

### 1.1 历次感悟迭代
- **功能**：记录对同一案例多次阅读的不同感悟
- **操作**：允许添加、修改、删除感悟
- **时间记录**：
  - 记录创建时间
  - 记录最后修改时间
- **使用场景**：
  - 第1次看案例：初步印象和发现
  - 第2次看案例：有了新的理解和视角
  - 第3次看案例：更深层的洞察
  - 随时可以修改或删除之前的感悟

### 1.2 标签管理
- **功能**：在案例详情页直接修改该案例的标签
- **操作位置**：案例详情页的标签显示区域
- **使用场景**：
  - 看到案例标签分配不对
  - 直接在案例详情页手工修改
  - 添加缺失的标签
  - 删除不准确的标签
- **注意**：不是全局标签库管理，而是针对单个案例的标签调整

### 1.3 案例内容编辑（新增）
- **功能**：直接编辑案例的各个字段信息
- **可编辑内容**：
  - 基本信息（代号、性别、年龄、职业、婚姻状况）
  - 接访信息（日期、时长、渠道、次数）
  - 危机评估（危机等级、证据）
  - 关键词列表
  - 使用技术列表
  - AI分析内容（摘要、优势、改进建议、下次建议）
  - 完整对话记录
- **使用场景**：
  - 发现案例信息有遗漏，需要补充
  - 对话记录有错误，需要修正
  - AI分析不准确，需要手工调整
  - 补充遗漏的关键词或技术

### 1.4 模拟对话（预留设计）
- **功能**：基于真实案例进行角色扮演式的模拟对话练习
- **使用场景**：
  - 学习者扮演咨询师，与AI扮演的来访者对话
  - 基于案例背景，练习不同的干预技术
  - 尝试不同的应对策略
  - 获得即时反馈和建议
- **数据存储**：
  - 每次模拟对话作为独立记录保存
  - 关联到原始案例
  - 记录对话内容、使用的技术、AI反馈
- **注意**：此功能在后续开发，当前方案需预留数据结构

#### 4.2.2 创建处理脚本

**文件：`src/process_updates.py`**

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
处理前端提交的更新请求
功能：处理感悟和标签的增删改操作
"""

import json
from datetime import datetime
from pathlib import Path
import sys

PROJECT_ROOT = Path(__file__).parent.parent
DATA_DIR = PROJECT_ROOT / "data"
CASES_DIR = DATA_DIR / "cases" / "processed"
PENDING_FILE = DATA_DIR / "pending_updates.json"

def generate_id(prefix='sr'):
    """生成唯一ID"""
    timestamp = datetime.now().strftime('%Y%m%d%H%M%S')
    return f"{prefix}_{timestamp}"

def add_supervision(case_id, content):
    """添加感悟"""
    case_file = CASES_DIR / f"{case_id}.json"
    
    with open(case_file, 'r', encoding='utf-8') as f:
        case_data = json.load(f)
    
    if 'supervision_records' not in case_data:
        case_data['supervision_records'] = []
    
    now = datetime.now().isoformat()
    record = {
        'id': generate_id('sr'),
        'created_at': now,
        'updated_at': now,
        'content': content
    }
    
    case_data['supervision_records'].append(record)
    
    with open(case_file, 'w', encoding='utf-8') as f:
        json.dump(case_data, f, ensure_ascii=False, indent=2)
    
    print(f"✅ 已添加感悟到案例 {case_id}")

def edit_supervision(case_id, record_id, content):
    """编辑感悟"""
    case_file = CASES_DIR / f"{case_id}.json"
    
    with open(case_file, 'r', encoding='utf-8') as f:
        case_data = json.load(f)
    
    for record in case_data.get('supervision_records', []):
        if record['id'] == record_id:
            record['content'] = content
            record['updated_at'] = datetime.now().isoformat()
            break
    
    with open(case_file, 'w', encoding='utf-8') as f:
        json.dump(case_data, f, ensure_ascii=False, indent=2)
    
    print(f"✅ 已更新感悟 {record_id}")

def delete_supervision(case_id, record_id):
    """删除感悟"""
    case_file = CASES_DIR / f"{case_id}.json"
    
    with open(case_file, 'r', encoding='utf-8') as f:
        case_data = json.load(f)
    
    case_data['supervision_records'] = [
        r for r in case_data.get('supervision_records', [])
        if r['id'] != record_id
    ]
    
    with open(case_file, 'w', encoding='utf-8') as f:
        json.dump(case_data, f, ensure_ascii=False, indent=2)
    
    print(f"✅ 已删除感悟 {record_id}")

def update_tags(case_id, tags):
    """更新标签"""
    case_file = CASES_DIR / f"{case_id}.json"
    
    with open(case_file, 'r', encoding='utf-8') as f:
        case_data = json.load(f)
    
    case_data['tags'] = tags
    
    with open(case_file, 'w', encoding='utf-8') as f:
        json.dump(case_data, f, ensure_ascii=False, indent=2)
    
    print(f"✅ 已更新案例 {case_id} 的标签")

def process_pending_updates():
    """处理待处理的更新"""
    if not PENDING_FILE.exists():
        return
    
    with open(PENDING_FILE, 'r', encoding='utf-8') as f:
        data = json.load(f)
    
    for update in data.get('updates', []):
        action = update['action']
        case_id = update['case_id']
        
        if action == 'add_supervision':
            add_supervision(case_id, update['data']['content'])
        elif action == 'edit_supervision':
            edit_supervision(case_id, update['data']['record_id'], update['data']['content'])
        elif action == 'delete_supervision':
            delete_supervision(case_id, update['data']['record_id'])
        elif action == 'update_tags':
            update_tags(case_id, update['data']['tags'])
    
    # 清空待处理文件
    with open(PENDING_FILE, 'w', encoding='utf-8') as f:
        json.dump({'updates': []}, f)
    
    # 重新生成案例库
    print("\n🔄 重新生成案例库...")
    import subprocess
    subprocess.run([sys.executable, 'src/generate_case_library.py'])

if __name__ == '__main__':
    process_pending_updates()
```

## 五、完整操作流程

### 5.1 添加感悟流程

1. 用户打开案例详情页
2. 滚动到"历次感悟"区域
3. 点击"+ 添加新感悟"按钮
4. 在模态框中输入感悟内容
5. 点击"保存"
6. JavaScript将数据写入 `pending_updates.json`
7. 页面提示"正在保存，请手动运行：python src/process_updates.py"
8. 用户运行命令，Python处理更新并重新生成HTML
9. 刷新页面，查看新添加的感悟

### 5.2 编辑/删除感悟流程

1. 在感悟卡片上点击"编辑"或"删除"
2. 编辑：弹出模态框，修改内容后保存
3. 删除：确认后直接删除
4. 同样写入 `pending_updates.json`
5. 运行 `python src/process_updates.py` 处理

### 5.3 修改标签流程

1. 在案例详情页点击"编辑标签"
2. 进入编辑模式：
   - 每个标签显示"×"删除按钮
   - 底部显示"添加标签"按钮
3. 删除不正确的标签（点×）
4. 添加新标签：
   - 点击"添加关系标签"或"添加症状标签"
   - 从标签库中选择
5. 点击"保存"
6. 写入 `pending_updates.json`
7. 运行 `python src/process_updates.py` 处理
