## 一、项目概述

### 1.1 项目目标

构建一个**本地化、AI 驱动的心理咨询案例管理系统**，实现：
1. **接访记录（Word）** → **AI 处理（JSON）** → **案例库整理（HTML）**
2. 智能分析：AI 自动提取标签、关键词、危机等级、技术应用
3. 多维检索：按关系标签、精神症状、危机等级、关键词快速查询
4. 可视化界面：案例导入、查询、展示的 Web 界面

### 1.2 三层架构

```
层次1：原始接访记录（Word）
    ↓ AI 处理
层次2：结构化案例（JSON + 四大索引）
    ↓ 知识库整理
层次3：案例库（HTML 可视化）
```

## 九、关键决策记录

### 9.1 为什么选择三层架构？

- **层次1（Word）**：保留原始记录，作为法律证据，永不修改
- **层次2（JSON）**：结构化数据，便于程序处理和索引
- **层次3（HTML）**：可视化展示，便于人类阅读和学习

### 9.2 为什么不用数据库？

- 案例量预计 < 500，JSON 足够
- 易于备份和迁移
- 易于手动修改和校对
- 无需额外的数据库服务

### 9.3 为什么选择 HTML 而非 Obsidian？

- HTML 可在任何浏览器打开，无需额外软件
- 可定制化程度高
- 便于部署和分享
- 如果案例量增长，后续可迁移到 Obsidian

## 十、风险与挑战

### 10.1 AI 分析准确性

**风险**：AI 提取的标签、危机等级可能不准确

**应对**：
- 所有 AI 分析结果需要人工复核
- 提供手动修改 JSON 的机制
- 逐步优化 AI 提示词

### 10.2 数据隐私

**风险**：案例包含敏感个人信息

**应对**：
- 所有数据本地存储，不上传云端
- 案主代号脱敏处理
- 定期加密备份

### 10.3 案例量增长

**风险**：案例量超过 500 时，JSON 检索效率下降

**应对**：
- 当前阶段先用 JSON
- 达到 500 后考虑迁移到 SQLite
- 或迁移到 Obsidian + Dataview
