## 详细记录

### 1. 问题诊断
**现象**：流派分析的编辑/上传/下载按钮点击无效
**错误**：`Uncaught ReferenceError: switchTab is not defined`

**原因排查**：
```bash
grep -n "function switchTab" visit_001.html    # 第1845行
grep -n "onclick=\"switchTab" visit_001.html   # 第216行
```
发现：按钮在216行调用，函数在1845行才定义

### 2. 根本原因
HTML生成顺序问题：
1. `get_html_template()` → head + body开始
2. 页面内容生成 → 包含按钮的onclick属性
3. `get_html_footer()` → 定义JS函数 + 结束标签

**问题**：inline事件处理器（onclick）要求函数在调用前已定义

### 3. 解决方案
**重构模板结构**：
```python
# 修改前
def get_html_template():
    return """<head>...</head><body>"""

def get_html_footer():
    return """<script>所有函数...</script></body></html>"""

# 修改后  
def get_html_template():
    return """<head>
        <script>所有函数定义...</script>
    </head><body>"""

def get_html_footer():
    return """</body></html>"""
```

## 技术要点

### HTML/JS加载顺序原则
- ✅ 函数先定义，后调用
- ✅ JS放在`<head>`中或使用事件监听器
- ❌ 不要在底部定义函数，顶部就调用

### 为什么每次都出现
- 不是"按钮丢失"，而是生成逻辑问题
- 每次运行脚本都会按错误顺序生成
- 必须修复模板，而非手动改HTML
