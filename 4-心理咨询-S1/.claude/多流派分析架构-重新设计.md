# 多流派分析架构 - 重新设计

## 一、核心洞察

### 1.1 打破"大观学派"的限制

**原有思维局限**：
- 当前架构将"大观学派"硬编码到目录结构中（`output/大观学派/`）
- 标签体系、危机等级、技术列表都绑定大观学派
- 一个案例只有一套分析

**新的认知**：
- **案例是中性的**：案例就是案例，不属于任何流派
- **分析是多元的**：同一个案例可以用不同流派分析
- **学习是迭代的**：随着能力成长，对同一案例的理解会加深

### 1.2 架构重构方向

```
原架构：
案例 → 大观学派分析 → HTML展示

新架构：
              ┌→ 大观学派分析（危机干预视角）
              │
案例（中性）  ┼→ CBT分析（认知行为视角）
              │
              ├→ 精神动力学分析（潜意识视角）
              │
              ├→ 人本主义分析（来访者中心）
              │
              └→ 存在主义分析（意义探索）
```

### 5.2 迁移策略

**方案A：原地升级（推荐）**

创建迁移脚本 `src/migrate_to_multiapproach.py`：

```python
def migrate_case(old_json):
    """将单流派JSON迁移为多流派JSON"""
    
    # 提取流派无关的基础信息
    new_json = {
        "case_id": old_json["case_id"],
        "source_file": old_json["source_file"],
        "created_at": old_json["created_at"],
        "version": "2.0",  # 升级版本号
        "last_modified": datetime.now().isoformat(),
        
        "basic_info": old_json["basic_info"],
        "session_info": old_json["session_info"],
        "dialogue": old_json["dialogue"],
        
        # 将原有分析移到 analyses.daguanpai
        "analyses": {
            "daguanpai": {
                "approach": "大观学派",
                "created_at": old_json["created_at"],
                "updated_at": old_json["created_at"],
                "analyst": "AI + 人工审核",
                
                "crisis_assessment": {
                    "level": old_json["crisis_level"],
                    "evidence": old_json["crisis_evidence"]
                },
                
                "tags": old_json["tags"],
                "keywords": old_json["keywords"],
                "techniques_used": old_json["techniques_used"],
                "analysis": old_json["ai_analysis"]
            }
        },
        
        "supervision_records": old_json.get("supervision_records", []),
        "simulation_sessions": [],
        "related_cases": []
    }
    
    return new_json
```

**迁移步骤**：
1. 备份现有JSON到 `data/backup/`
2. 运行迁移脚本
3. 验证新JSON结构
4. 更新HTML生成脚本

### 5.3 向后兼容

为避免大规模重构，可以提供兼容层：

```python
def load_case_compatible(case_id):
    """加载案例，自动兼容旧格式"""
    with open(f"data/cases/processed/{case_id}.json") as f:
        data = json.load(f)
    
    # 检测版本
    if "version" not in data or data["version"] == "1.0":
        # 旧格式，自动转换
        return migrate_case(data)
    else:
        # 新格式，直接返回
        return data
```

**Q3：感悟如何与流派关联？**

在 `supervision_records` 中添加 `approach` 字段：

```json
{
  "id": "sr_001",
  "approach": "大观学派",  // 标注视角
  "content": "..."
}
```

在UI中可以筛选：
- 查看所有感悟
- 只看"大观学派"视角的感悟
- 只看"CBT"视角的感悟
