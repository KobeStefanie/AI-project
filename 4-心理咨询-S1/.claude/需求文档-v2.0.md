## §1 项目概述

### §1.1 项目目标

构建一个**本地化、AI 驱动的心理咨询案例管理系统**，实现：

1. **案例归档**：Word 文档 → 结构化 JSON + 四大索引
2. **智能分析**：AI 自动提取标签、关键词、危机等级、技术应用
3. **多维检索**：按关系标签、精神症状、危机等级、关键词快速查询
4. **督导管理**：一个案例支持多个督导记录（一对多关系）
5. **可视化界面**：案例导入、查询、展示的 Web 界面

**核心原则**：
- 纯本地运行，数据不出本机
- AI 辅助而非替代，最终由咨询师确认
- 原始记录永久保存，AI 处理后的结果单独存储（一一对应）
- 随用户能力成长而进化（大观 → CBT → 精神动力学 → ...）

### §1.2 用户能力画像

| 体系 | 进度 | 具体内容 |
|------|------|----------|
| 大观学派 | 已学，实操中 | 希望热线危机干预（五法宝、六变三托、危机等级判读、逆向倒推、同步同理心、非事件原则） |
| CBT | 零基础，近期开始 | 待启动 |
| 精神动力学 | 未开始 | 后续学习计划 |
| 人本主义 | 未开始 | 后续学习计划 |

## §3 核心功能

### §3.1 案例导入（已完成 UI）

**位置**：`接访记录模板-v1.0.html` 顶部"案例导入"区

**功能**：
1. **方式一**：拖拽 Word 文档到上传区
2. **方式二**：点击上传区选择文件
3. **方式三**：粘贴文件路径 + 点击"处理"按钮

**支持格式**：`.docx`（不支持 `.doc`）

**文件命名建议**：`YYYYMMDD-案例名称.docx`（如：`20260616-善良女孩的求助.docx`）

**处理流程**：
- 用户上传/输入路径后，界面提示："发送给 AI：处理这个案例：路径"
- AI 调用 `case_processor.py` 脚本
- 脚本自动生成案例编号、提取信息、AI 分析、更新索引

### §3.2 案例编号系统

**格式**：`C + YYYYMMDD + 3位序号`

**示例**：
- `C20260616001` - 2026年6月16日第1个案例
- `C20260616002` - 2026年6月16日第2个案例
- `C20260616003` - 2026年6月16日第3个案例

**生成规则**：
1. 读取当天已有的最大序号
2. 自动递增（001 → 002 → 003）
3. 跨天后序号重置为 001

**用途**：
- 作为文件名前缀（Word、JSON、督导记录）
- 作为唯一标识符（索引、查询、关联）

#### 3.3.3 危机等级索引（crisis_levels）

**分级标准**（大观学派希望热线体系）：

| 等级 | 名称 | 标准 | 干预措施 |
|------|------|------|----------|
| **S** | 自杀风险 | 有明确计划/手段/遗书/失联 | 立即报警/紧急联系人/120 |
| **L** | 生命危险 | 他杀风险/严重自伤/精神病性症状 | 24小时内精神科就诊 |
| **M** | 中度危机 | 抑郁/焦虑影响功能/创伤急性期 | 72小时内复访/药物评估 |
| **C** | 慢性困扰 | 长期关系问题/自我成长议题 | 常规咨询频率 |
| **Z** | 正常范围 | 偶发情绪波动/一般性困惑 | 心理教育/自助资源 |

### §3.4 AI 自动分析

**调用接口**：CatKingAI API（自研网关，默认首选）

**分析内容**：
1. **案例摘要**（200字以内）
2. **咨询师优势**（做得好的地方，3-5条）
3. **改进建议**（可以提升的地方，3-5条）
4. **下次咨询建议**（具体方向和策略）

**输出格式**：
```json
{
  "ai_analysis": {
    "summary": "16岁女孩因与同学关系破裂出现自杀风险...",
    "strengths": [
      "快速识别 S 级危机，立即启动六变三托筛查",
      "使用同步同理心建立信任，避免说教",
      "及时引入资源（父母、老师），打破孤立感"
    ],
    "improvements": [
      "可更早探索'是否告诉父母'的矛盾心理",
      "五法宝的'打包'步骤可以更充分",
      "结束时的安全承诺可以更具体"
    ],
    "recommended_followup": "24小时内电话回访确认安全；下次咨询探索人际冲突背后的自我价值感议题；评估是否需要精神科药物支持"
  }
}
```

**注意事项**：
- AI 分析结果仅供参考，最终判断由咨询师确认
- 如果 AI 分析明显错误，咨询师可手动修改 JSON 文件
- 危机等级判断必须由咨询师复核

### §3.5 督导记录管理（一对多）

**命名规则**：`案例编号_S序号.md`

**示例**：
```
C20260616003_S001.md  - 第1次督导（使用大观学派督导）
C20260616003_S002.md  - 第2次督导（使用 CBT 督导）
C20260616003_S003.md  - 第3次督导（使用精神动力学督导）
```

**关联关系**：
- 同一案例可以有多次督导
- 不同督导可能使用不同理论取向
- 督导记录独立存储，不修改原案例 JSON

## §4 接访记录模板字段规范

### §4.1 字段分类

| 类型 | 说明 | 示例字段 |
|------|------|----------|
| **必填** | 不填无法生成有效记录 | 案主代号、接访日期、对话内容 |
| **选填** | 可以留空，根据实际情况填写 | 电话、地址、家庭成员、服药情况 |
| **AI自动填充** | 从对话内容自动提取 | 关系标签、精神症状标签、危机等级、关键词、使用的技术、AI分析 |

### §4.2 具体字段列表

#### Zone 0 - 案例导入
- 文件上传/路径输入（触发 AI 处理）

#### Zone 1 - 基本信息
| 字段 | 类型 | 说明 |
|------|------|------|
| 案主代号 | 必填 | 格式：姓氏+XX（如：张XX） |
| 接访次数 | 必填 | 第X次 |
| 当前阶段 | 选填 | 如：阶段1-婆媳关系+焦虑症 |
| 接访日期 | 必填 | YYYY-MM-DD |
| 通话时长 | 必填 | 如：90分钟 |
| 咨询渠道 | 必填 | 电话/视频/面对面 |

#### Zone 2 - 个人资料（选填）
- 性别、年龄、性取向、职业、婚姻状况、子女情况、学历、宗教
- 联系电话、地址、紧急联系人（隐私字段，加密存储）

#### Zone 3 - 精神科背景（选填）
- 精神科就医史、服药情况、既往咨询/住院史

#### Zone 4 - 家庭信息（选填）
- 家庭成员、原生家庭情况

#### Zone 5 - 大观量化（接访后马上填）
- 危机等级判读 + 判据
- 六变三托信号筛查
- 五法宝步骤自检
- 逆向倒推路径
- 求助动机/等级评估
- 主诉 SOR 转码

#### Zone 6 - 对话内容（必填）
- 完整的咨询对话记录

#### Zone 7 - CBT 分析（待用户学 CBT 后启用，选填）
- 自动思维识别、认知歪曲类型、核心认知重构点、转折之语

#### Zone 8 - 策略与反思
- 使用的技术及效果
- 自我状态检查
- 下次咨询目标与策略

#### AI 自动字段（不在模板中，由脚本生成）
- 关系标签（relation_tags）
- 精神症状标签（symptom_tags）
- 危机等级（crisis_levels）
- 关键词（keywords）
- AI 分析（summary, strengths, improvements, recommended_followup）

### §4.3 原始记录 vs 处理后记录

| 维度 | 原始记录 | 处理后记录 |
|------|----------|------------|
| **格式** | Word (.docx) | JSON |
| **存储位置** | data/cases/original/ | data/cases/processed/ |
| **文件名** | C20260616003.docx | C20260616003.json |
| **用途** | 法律证据、人类阅读 | 程序查询、数据分析 |
| **修改** | 永不修改 | 可修改（如 AI 分析错误） |
| **对应关系** | 1:1 严格对应 | 1:1 严格对应 |

✅ **完整性**：原始记录与处理后记录 1:1 对应，无遗漏  
✅ **可扩展性**：支持新增理论取向（CBT、精神动力学等）
