# 心理咨询案例管理系统 - Phase 1 完成

## 🎉 已完成功能

### 1. 数据结构设计 ✅
- 完整的JSON数据结构（静态信息 + 动态信息 + 会谈记录）
- 支持案例状态流转：active → closed → archived
- 咨询目标和用药情况支持修改历史记录
- 会谈记录包含300字概要（AI生成）

### 2. 后端API服务 ✅
**文件**: `src/case_api.py`

**接口列表**:
- `GET  /api/cases` - 获取所有案例列表
- `POST /api/cases` - 创建新案例（首次接访）
- `GET  /api/cases/{case_id}` - 加载指定案例
- `POST /api/cases/{case_id}/sessions` - 添加新会谈记录
- `PUT  /api/cases/{case_id}/sessions/{session_num}` - 修改会谈记录
- `PUT  /api/cases/{case_id}/goal` - 更新咨询目标
- `PUT  /api/cases/{case_id}/medication` - 更新用药情况
- `PUT  /api/cases/{case_id}/close` - 结案
- `POST /api/ai/summary` - 生成会谈概要（AI）

**特性**:
- 自动备份（每次保存前备份旧数据）
- 年龄自动计算（根据出生日期）
- 会谈次数自动递增
- 支持跨状态查询（active/closed）

### 3. 案例列表页面 ✅
**文件**: `output/接访记录/case-list.html`

**功能**:
- 显示所有案例（进行中 + 已结案）
- 统计卡片（进行中数量、已结案数量、总数）
- 状态筛选（全部/进行中/已结案）
- 搜索功能（代号、主诉）
- 点击卡片跳转到接访记录表单

### 4. 接访记录表单（升级版）✅
**文件**: `output/接访记录/intake-record-new.html` + `intake-record-new.js`

**新增功能**:
- **案例加载区域**：输入案例代号，加载已有案例进行后续会谈
- **两种工作模式**：
  - 首次接访：填写完整信息，创建新案例
  - 后续会谈：加载案例，只填写本次新内容
  
**后续会谈模式特性**:
- 基本信息、主诉、家庭结构、既往史 → 只读显示
- 咨询目标、用药情况 → 可修改（记录历史）
- 接访次数 → 自动递增
- 显示上次会谈概要（黄色高亮区域）
- 隐藏Word导入区域

## 🚀 使用说明

### 启动系统

双击 `启动服务.bat`，系统会自动：
1. 启动后端API服务器（端口 5000）
2. 启动前端服务器（端口 8888）
3. 打开浏览器访问案例列表

或者手动启动：
```bash
# 终端1：启动后端API
cd D:\AI-项目\4-心理咨询-S1
python src\case_api.py

# 终端2：启动前端服务
cd D:\AI-项目\4-心理咨询-S1\output\接访记录
python -m http.server 8888
```

### 首次接访流程

1. 打开案例列表 → 点击"新建案例"
2. 填写完整表单（所有必填字段）
3. 点击"保存案例"创建新案例
4. 系统生成案例JSON文件到 `data/cases/active/`

### 后续会谈流程

1. 打开案例列表 → 点击案例卡片
   - 或者在接访记录表单顶部输入案例代号 → 点击"加载案例"
2. 系统自动填充基本信息（只读）
3. 显示上次会谈概要（黄色区域）
4. 填写本次会谈内容：
   - 对话记录
   - 标签选择
   - 危机评估
   - 使用技巧
   - 督导记录
5. 点击"保存会谈"添加到案例

## 📝 数据结构要点

### 静态信息（填一次）
- 代号、性别、出生日期、职业、婚姻状况
- 主诉
- 家庭结构
- 既往史

### 动态信息（可修改，记录历史）
- 咨询目标（含修改历史）
- 用药情况（含修改历史）
- 紧急联系人、联系电话
- 来访备注

### 会谈记录（每次一条）
- 会谈次数、日期、咨询师、时长
- 上次会谈概要（自动加载）
- 本次对话记录
- 本次会谈概要（AI生成300字）
- 标签、危机评估、关键词、使用技巧
- AI分析、督导记录

## 🎯 关键设计决策

1. **案例代号唯一性**：系统在创建时检查代号是否已存在
2. **年龄自动计算**：根据出生日期自动计算当前年龄
3. **会谈次数自动递增**：无需手动填写
4. **上次概要自动加载**：后续会谈时自动显示上次的summary
5. **静态信息只读**：后续会谈时不可修改基本信息
6. **动态信息可修改**：咨询目标、用药情况可更新，记录历史
7. **自动备份**：每次保存前自动备份到 `data/backup/`

## ⚠️ 注意事项

1. **首次使用**：确保安装 Flask 和 flask-cors
   ```bash
   pip install flask flask-cors
   ```

2. **端口冲突**：如果5000或8888端口被占用，需要修改：
   - 后端API：修改 `src/case_api.py` 第行 `app.run(port=5000)`
   - 前端服务：修改启动命令的端口号
   - JS中的API地址：修改 `intake-record-new.js` 第6行 `CASE_API`

3. **数据安全**：
   - 案例数据存储在本地JSON文件
   - 每次修改自动备份
   - 建议定期备份整个 `data/` 目录

4. **多次来访场景**：
   - 10-20次会谈是常态
   - 基本信息只填一次，避免重复劳动
   - 咨询目标中途可能调整（记录原因）

## 📞 问题排查

### API连接失败
- 检查后端服务是否启动（端口5000）
- 浏览器控制台查看错误信息
- 检查防火墙是否拦截

### 案例加载失败
- 确认案例代号是否正确
- 检查 `data/cases/active/` 目录下是否有对应JSON文件
- 查看后端控制台的错误日志

### 表单字段未填充
- 检查浏览器控制台的JavaScript错误
- 确认JSON数据结构完整
- 检查字段ID是否匹配
