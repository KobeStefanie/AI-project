# 心理咨询案例系统 - 资料管理

## 📋 功能概述

**统一接访记录管理器** - 一个界面管理所有案例资料

通过统一的接访记录管理界面，可以为每个案例完整地管理：

### 1. 🎙️ 录音管理
- 上传案例咨询录音文件（支持 MP3/WAV/M4A/OGG/AAC）
- 拖拽上传 + 文件选择
- 在线播放录音
- 自动关联到对应案例

### 2. 📝 逐字稿管理
- 添加/编辑/删除咨询对话逐字稿
- 可选关联录音文件
- 多版本逐字稿管理
- 支持长文本编辑

### 3. 📋 督导资料管理
- 上传督导笔记、讨论记录等文档
- 支持多种格式（TXT/MD/PDF/DOC/DOCX/JPG/PNG）
- 添加标题和备注说明
- 便捷下载查看

### 4. 📊 资料汇总
- 实时统计录音、逐字稿、督导资料数量
- 一键查看案例库
- 完整的工作流指引

## 🎯 推荐工作流

### 完整的案例管理流程

1. **打开接访记录管理器**
   - 访问：`output/接访记录/intake-record-manager.html`
   - 或从项目首页点击"接访记录管理"

2. **输入案例编号**
   - 例如：`C20260616001`
   - 点击"加载案例"

3. **上传录音** (Tab 1)
   - 拖拽或选择录音文件
   - 自动上传并显示播放器

4. **添加逐字稿** (Tab 2)
   - 输入或粘贴对话内容
   - 可选择关联录音文件
   - 保存后可随时编辑

5. **上传督导资料** (Tab 3)
   - 填写资料标题和备注
   - 上传督导文档
   - 支持多个文件

6. **查看汇总** (Tab 4)
   - 查看所有资料统计
   - 一键打开案例库
   - 确认资料完整性

7. **案例库查看**
   - 所有资料自动显示在案例详情页
   - 录音可在线播放
   - 逐字稿折叠展示
   - 督导资料可下载

## 💡 使用方式

### 方式一：统一管理器（推荐）

**入口**：`output/接访记录/intake-record-manager.html`

✅ **优势**：
- 所有功能集中在一个界面
- Tab切换，操作流畅
- 实时统计，进度可视
- 一个案例一次性完成所有资料采集

- 督导资料：`output/接访记录/supervision-manage.html`

## 🔌 API接口

### 录音管理 API (端口 8004)
- `GET  /api/audio/list?case_id=xxx` - 获取案例录音列表
- `POST /api/audio/upload` - 上传录音（multipart/form-data）
- `GET  /api/audio/file/{case_id}/{filename}` - 播放/下载录音
- `DELETE /api/audio/delete` - 删除录音

### 逐字稿管理 API (端口 8005)
- `GET  /api/transcript?case_id=xxx` - 获取案例逐字稿
- `POST /api/transcript` - 添加逐字稿
- `PUT  /api/transcript` - 更新逐字稿
- `DELETE /api/transcript` - 删除逐字稿

### 督导资料管理 API (端口 8006)
- `GET  /api/supervision/list?case_id=xxx` - 获取督导资料列表
- `POST /api/supervision/upload` - 上传督导资料（multipart/form-data）
- `GET  /api/supervision/file/{case_id}/{filename}` - 下载督导资料
- `DELETE /api/supervision/delete` - 删除督导资料

## 💡 使用流程

1. **录音上传**：在接访记录管理器上传咨询录音
2. **逐字稿整理**：添加逐字稿并关联录音文件
3. **督导资料**：上传督导笔记和讨论记录
4. **查看汇总**：确认所有资料完整
5. **案例库查看**：案例库自动显示所有关联资料
6. **资料归档**：所有资料按案例编号自动分类存储

## 🔧 技术架构

### 服务端口分配
- **8003** - 流派配置服务
- **8004** - 录音管理服务
- **8005** - 逐字稿管理服务
- **8006** - 督导资料管理服务

### 数据存储
- **录音文件**：`data/cases/audio/{case_id}/`
- **逐字稿**：案例JSON的 `transcripts` 字段
- **督导资料**：`data/cases/supervision/{case_id}/`
- **案例元数据**：`data/cases/processed/{case_id}.json`

## 📊 案例库集成

案例详情页按以下顺序显示资料：

1. **基本信息** - 案例元数据
2. **🎙️ 录音资料** - 内嵌音频播放器
3. **📝 逐字稿** - 可折叠展开查看
4. **📋 督导资料** - 文件列表和下载链接
5. **流派分析** - 多流派Tab分析内容

## ⚠️ 注意事项

- 所有服务必须启动才能使用完整功能
- 录音和督导文件直接存储在文件系统
- 逐字稿存储在案例JSON中，适合文本内容
- 案例JSON会自动更新 `last_modified` 时间戳
- 删除操作不可恢复，请谨慎操作

## 🔄 重新生成案例库

修改资料后需要重新生成案例库HTML：

```bash
python src/generate_case_library.py
```

案例JSON更新会自动反映，但HTML需要手动重新生成。
