## 🎯 使用场景

### 场景 1：新案例录入（完整流程）
1. 打开统一管理器
2. 输入案例编号（如 C20260627001）
3. 上传录音 → 添加逐字稿 → 上传督导资料
4. 查看汇总确认完整
5. 案例库自动显示所有资料

### 场景 2：补充现有案例资料
1. 打开统一管理器
2. 输入已有案例编号
3. 切换到对应Tab补充资料
4. 资料自动关联到案例

### 场景 3：仅上传录音
- 使用独立的录音上传界面
- 或使用统一管理器的录音Tab

## 🔄 工作流集成

### 案例完整生命周期

```
1. 接访咨询
   ↓
2. 录音记录
   ↓
3. 整理逐字稿
   ↓
4. 督导讨论 → 上传督导资料
   ↓
5. AI处理（案例处理脚本）
   ↓
6. 生成案例库
   ↓
7. 案例学习与查阅
```

**资料管理系统负责步骤 2-4**，与现有系统无缝衔接。

## ✨ 核心优势

### 1. 一体化管理
所有案例资料在一个界面完成采集，避免来回切换。

### 2. 自动关联
所有上传的资料自动关联到案例，无需手动维护关系。

### 3. 实时同步
资料上传后立即保存到案例JSON，重新生成案例库后即可查看。

### 4. 多种格式支持
- 录音：MP3/WAV/M4A/OGG/AAC
- 文档：TXT/MD/PDF/DOC/DOCX
- 图片：JPG/PNG

### 5. 进度可视化
资料汇总Tab实时显示采集进度，一目了然。

## ⚠️ 注意事项

1. **服务器必须启动**：所有功能依赖后端服务，使用前确保服务已启动
2. **案例库更新**：上传资料后需重新生成案例库HTML才能在案例详情页看到
3. **文件大小**：建议单个录音文件不超过100MB，督导资料不超过20MB
4. **浏览器兼容**：建议使用Chrome/Edge/Firefox最新版
5. **数据安全**：删除操作不可恢复，请谨慎操作

## 🔧 故障排查

### 问题1：无法上传文件
- 检查对应服务是否启动（8004/8005/8006）
- 查看浏览器控制台是否有CORS错误
- 确认案例编号已输入

### 问题2：录音无法播放
- 检查录音格式是否支持
- 确认audio_server.py正在运行（端口8004）
- 检查文件路径是否正确

### 问题3：案例库不显示新上传的资料
- 运行 `python src/generate_case_library.py` 重新生成
- 刷新浏览器清除缓存（Ctrl+F5）
