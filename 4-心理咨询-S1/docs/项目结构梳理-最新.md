## 三、用户真实需求（根据最后确认）

### 完整工作流
```
1. 打开Web界面"单案例处理"
2. 上传Word接访记录
3. AI自动分析（标签、危机等级、关键词等）
4. 人工在界面上核实、修改信息
5. 上传录音、逐字稿、督导资料
6. 点击"保存"，后台自动：
   - 保存到 data/cases/processed/{案例编号}/
   - 更新四大索引
   - 案例库自动包含新案例
```

### 目录结构要求
```
data/cases/processed/
└── C20260627001/              # 同一案例文件夹
    ├── C20260627001.json      # 案例数据
    ├── audio/                 # 录音子目录
    │   ├── 录音1.mp3
    │   └── 录音2.m4a
    ├── supervision/           # 督导资料子目录
    │   ├── 督导笔记.md
    │   └── 督导报告.pdf
    └── transcripts/           # 逐字稿子目录（可选）
```

### 问题2：系统功能重复

**intake-record-manager.html（昨天的）**：
- 功能：管理**已有案例**的录音、逐字稿、督导资料
- 前提：案例已经存在于 `processed/` 目录
- 操作：选择案例ID → 上传材料

**case-processor.html（今天的）**：
- 功能：**创建新案例**的完整流程
- 前提：上传Word文档开始
- 操作：上传Word → AI分析 → 核实 → 上传材料 → 保存

**结论**：两者功能不同，应该共存！
