# 需求文档

> 版本 v0.1 | 2026-06-13

## §1 项目概述

### §1.1 项目目标

将用户学到的心理咨询技能"蒸馏"为一个 Claude Code Skill，兼具两种身份：

- **督导**：接访后按用户的标准复盘、找盲点、给反馈
- **技能**：接访前/中辅助个案概念化、状态分析、干预策略制定

核心原则：**纯个人使用，高度私人化定制，随用户能力成长而成长。**

### §1.2 用户当前能力画像

| 体系 | 进度 | 具体内容 |
|------|------|----------|
| 大观学派 | 已学，实操中 | 希望热线危机干预话术（五法宝、六变三托、危机等级判读、逆向倒推、同步同理心、非事件原则） |
| CBT | 零基础，近期开始 | 待启动 |

### §1.3 扩展路线

CBT → 精神动力学 → 人本主义 → 存在主义（按学习进度逐一注入）

## §2 现状与痛点

### §2.1 接访记录诊断

已读取用户全部 9 份历史接访记录（路径：`D:\心理咨询师\1-接访记录\`）。三个发现：

1. **大观热线技能在记录中完全隐形**：用户已掌握五法宝、六变三托、危机等级判读、逆向倒推，但记录中无任何体现。原因不是不会，而是没有模板把训练落地到纸面。

2. **接访与复盘使用两套分离的语言**：接访时大观热线话术驱动（直觉、同步同理心），复盘时 CBT 框架驱动（认知歪曲、个案概念化）。两条线未汇合。

3. **缺的不是能力是结构**：6/10 的两个案例质量已有飞跃，细节丰富、反思深刻，但格式每次不同。

### §2.2 核心痛点

**没有一张表把大观热线训练和实际接访行为串起来。**

## §3 功能需求

### §3.1 接访记录模板

填空式，13 字段，四区：

**第一区：基本锚定**
- 日期、通话时长、次数、渠道

**第二区：大观量化（接访后马上填）**
1. 危机等级判读 + 判据
2. 六变三托信号筛查
3. 五法宝步骤自检
4. 逆向倒推路径
5. 求助动机/等级评估
6. 主诉 SOR 转码

**第三区：CBT 分析（复盘时填，待用户开始学 CBT 后启用）**
7. 自动思维识别
8. 认知歪曲类型
9. 核心认知重构点
10. 转折之语

**第四区：策略与反思**
11. 使用的技术及效果
12. 自我状态检查
13. 下次咨询目标与策略

### §3.2 Skill 督导模式

用户提供接访记录 → Skill 按大观标准复盘：
- 危机判读对不对
- 五法宝步骤缺了哪步
- 逆向倒推推到位了吗
- 记录规范性检查
- 给出具体改进建议

### §3.3 Skill 技能模式

用户描述来访者场景 → Skill 输出：
- 大观线：危机等级预判 + 逆向倒推话术草稿 + 六变三托筛查
- CBT 线（待注入后启用）
- 一页纸策略建议

### §3.4 成长机制

- Skill 文件内置「成长日志」区
- 用户每次学新内容 → 追加注入
- 每次接访新领悟 → 追加
- CBT 学完后 → 启用第三区字段 + 技能模式 CBT 线
- 后续流派 → 作为新模块叠加

## §6 设计原则

- **只写已掌握的**：不堆砌未学理论
- **填空代替写作**：接访后不是"写"记录，是"答"字段
- **大观先行**：大观是用户已有的肌肉记忆，模板把它显性化
- **可进化**：Skill 从 v0.1 开始，每次学习每次接访都是升级机会
- **纯个人使用**：不考虑分享给同行或学生
