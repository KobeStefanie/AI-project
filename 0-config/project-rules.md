# 项目目录规范

## 规则
- 凡是涉及使用 AI 开发项目的工作，项目文件必须统一放在 `D:\AI-项目\` 下
- 所有新项目统一存储在 `D:\AI-项目\` 下
- 项目根目录命名格式: `数字-名称`（如 `1-王子救公主`、`3-todo-app`）
- 数字按已有最大序号 +1
- 项目结构: 根目录放 README.md + 配置文件 + src/，源码全部进 src/
- 不得在 C:\ 下任何位置创建项目目录
- **⚡ 创建 AI 项目前必须先获得用户同意，未经明确许可不得擅自创建项目目录或初始化项目文件**

**Why:** 用户所有 AI 辅助项目集中管理，便于查找和维护。数字前缀保证排序和可追溯性。创建项目需经同意确保用户对项目有清晰认知和掌控。

**How to apply:** 用户说"新建项目/创建一个 XX"时，先列 D:\AI-项目\ 确定下一个序号，再按 `N-名称/src/` 结构创建。

## 命名反例
- ❌ `王子救公主`（缺数字前缀）
- ❌ `project1`（数字位置错）
- ❌ `1_王子救公主`（用 `_` 而非 `-`）
