# Windsurf 全局规则 · 个人项目规范

> 说明：本文件是可移植的全局规则副本。
> 在新账号 / 新机器上启用方法：将以下全部内容粘贴进
> `C:\Users\<用户名>\.codeium\windsurf\memories\global_rules.md`
> 或在 Windsurf 设置 → Rules → Global Rules 中粘贴保存。

---

## 1. 项目存储位置

- **所有由我创建的新项目，必须统一存储在 `D:\AI-项目\` 目录下。**
- 不得在 `C:\Users\...\CascadeProjects\`、桌面、文档等其他位置创建项目根目录。
- 若用户未显式指定路径，默认在 `D:\AI-项目\` 下新建项目文件夹。
- 若 `D:\AI-项目\` 不存在，须先创建该目录再建项目。

## 2. 项目命名格式

- **项目根目录命名格式必须为：`数字-名称`**
  - `数字`：递增的项目序号（1、2、3……），按 `D:\AI-项目\` 下已有的最大序号 +1。
  - `名称`：简洁的中文或英文项目名，描述项目用途。
- 示例：
  - ✅ `1-王子救公主`
  - ✅ `2-个人博客`
  - ✅ `3-todo-app`
  - ❌ `王子救公主`（缺少数字前缀）
  - ❌ `project1`（数字位置错误）
  - ❌ `1_王子救公主`（应使用 `-` 而非 `_`）

## 3. 项目内部结构

每个项目根目录下推荐包含：

```
数字-名称/
├── README.md          # 项目说明（用途、运行方式、目录结构）
└── src/               # 所有源代码归档于此
    └── ...
```

- 源代码统一放入 `src/` 子目录，不要散落在项目根目录。
- 项目根目录仅保留 `README.md`、配置文件（如 `package.json`、`.gitignore`）、`src/` 等顶层条目。

## 4. 创建新项目时的标准流程

当用户要求"新建项目 / 创建一个 XX"时，按以下步骤执行：

1. 检查 `D:\AI-项目\` 下已有项目，确定下一个可用序号 `N`。
2. 与用户确认项目名（或自行根据需求命名）。
3. 创建目录 `D:\AI-项目\N-名称\`。
4. 在其中创建 `README.md` 与 `src/` 子目录。
5. 把所有源代码文件写入 `src/`。
6. 完成后用绝对路径告知用户最终位置。

## 5. 禁止事项

- ❌ 不要在 `C:\` 下任何位置创建新的项目目录。
- ❌ 不要省略数字前缀。
- ❌ 不要把源代码直接放在项目根目录（除非该项目仅有单个 HTML 文件且用户明确要求）。
